## 1. 왜 지금 손봐야 하나 (현 상태 진단)

현재 분석 레이어는 "돌아가긴 하지만, 여론 수집 도구로서 인사이트가 거의 안 나오는" 상태다.
각 모듈을 코드·데이터 근거로 뜯어보면 다음과 같다.

### 1-1. 감성 분석 — 가장 취약 🔴
`analysis/sentiment.py`, `analysis/data/sentiment_ko.json`

| 문제 | 근거 | 결과 |
|---|---|---|
| **사전이 너무 작다** | 긍정 56 + 부정 63 + 부정어 7 = **126단어** | 실제 글·댓글의 대부분이 사전에 안 걸려 **'중립'으로 떨어짐** |
| **커뮤니티 은어·초성 미커버** | `ㄹㅇ, ㅅㅂ, 개꿀, 존나, 노잼, 꿀잼, ㅇㅈ, 씹덕, 갓-, -창, 좆-` 등 전무 | DCInside 여론의 실제 신호가 통째로 누락 |
| **강도(intensity) 없음** | "좋다" == "개좋다" == +1 | 온도차 반영 불가 |
| **3분류 라벨만** | pos/neg/neutral 카운트만 반환 | 연속 점수·신뢰도가 없어 시계열·집계 정밀도 낮음 |
| **이모티콘·반복표현 무시** | `ㅋㅋ, ㅠㅠ, ^^, !!!` 미처리 | 감정 신호 손실 |
| **반어/비꼼 불가** | 사전 방식의 원천 한계 | "참 잘하는 짓이다" → 긍정 오판 |

→ **"여론 분위기 지표"로서 신뢰하기 어렵다.** 개선 1순위.

### 1-2. 키워드 / 연관어 — 흔한 단어가 상위 독식 🟠
`analysis/keywords.py`

- **원시 빈도(raw count)만 센다** (`Counter.most_common`). 그래서 정보량 없는 흔한 명사(캐릭터·사람·생각·정도…)가 항상 상위 → **"이 갤/이 기간에 유독 튀는 단어(특징어)"를 못 뽑는다.**
- **TF-IDF / 로그오즈비(배경분포 대비) 부재** → 변별력 있는 키워드 추출 불가.
- **명사 단독(NNG/NNP/SL)만** 추출 → 복합 개념·고유명사구·bigram("○○ 업데이트", "○○ 캐릭터")을 못 잡는다.
- **불용어 52개**로 빈약 → 노이즈 다수.
- **연관어가 단순 동시출현 카운트** → 흔한 단어가 어떤 키워드에도 상위로 붙는다. **PMI/정규화 필요.**

### 1-3. 시계열 — '글 개수'만 센다 🟠
`analysis/timeseries.py`

- 날짜/시간/요일별 **글 수**만. **참여도(조회·추천·댓글) 추이 없음.**
- **요일 × 시간 히트맵 없음** → 활동 피크 파악이 약하다.
- **이슈 급상승(버스트) 탐지 없음** → *"오늘 이 갤에서 뭐가 터졌나"* 를 못 본다. 여론 수집 도구의 핵심 기능인데 빠져 있다.
- **감성 시계열 없음** → 분위기 변화 추적 불가.

### 1-4. 통계 / 작성자 — 얕다 🟡
`analysis/stats.py`

- 전체 카운트 + 단순 정렬 랭킹 + 카테고리 분포가 전부.
- **작성자 분석 없음**: 헤비 유저, 신규/복귀, 여론 주도층.
- **논쟁글(떡밥) 탐지 없음**: 추천≈비추 이거나 댓글이 폭발한 글 = 커뮤니티 이슈인데 못 짚는다.
- 참여 편중도(소수가 여론 주도하는지) 지표 없음.

### 1-5. 구조적 문제 🟡
- 모델/LLM 확장 훅이 `sentiment.py` **주석으로만** 존재. 실제 교체 가능한 추상화 레이어가 없다.
- **결과 캐싱·증분 없음** → 매 API 요청마다 전체 재계산 + kiwi 재토크나이즈(느림, 데이터 커지면 악화).

## 2. 개선 방향 (축별 제안)

### A. 감성/여론 분석 재설계 — 접근법 3안 비교

| 안 | 방식 | 장점 | 단점 | 비용 |
|---|---|---|---|---|
| **A1. 사전 보강** | 커뮤니티 슬랭 사전 + 강도 가중치 + 이모티콘 규칙 | 오프라인·무료·투명·빠름 | 반어 못 잡음, 유지보수 지속 필요 | 무료 |
| **A2. 한국어 감성모델** | KcELECTRA/KOTE 등 사전학습 모델 로컬 추론(감정 다분류 가능) | 슬랭에 강함, 문맥 반영 | 모델 다운로드·추론 자원, 파이프라인 무거움 | 무료(로컬) |
| **A3. LLM 배치 분석** | 글/댓글을 LLM로 감성+감정+요약 일괄 산출 | **정확도 최고**, 감성·토픽·요약을 한 번에, 반어까지 | API 키·비용·재현성 관리 필요 | 저렴* |

\* 현재 규모(420글/1,375댓글)면 **1회 전량 분석 비용이 극히 낮다**(수백 원~수천 원대). 증분 분석이면 더 저렴.

> **권장: A3를 주력 + A1을 오프라인 폴백.**
> 데이터가 작아 LLM 가성비가 압도적이고, 감성·토픽·요약을 한 파이프라인으로 묶을 수 있다.
> 단, API 없이도 도는 사전 기반 경로를 남겨 재현성·오프라인을 보장한다.
> `Scorer` 인터페이스로 추상화해 **런타임에 교체**(lexicon ↔ model ↔ llm) 가능하게 설계.

산출물: 문서별 **연속 감성점수(-1~+1)** + **신뢰도** + (LLM/모델 경로 시) **감정 다분류**(기쁨/분노/불안/혐오 등) → 집계·시계열에 그대로 재사용.

### B. 키워드 "특징어" 추출로 격상
- **TF-IDF** 또는 **weighted log-odds(배경분포 대비)** 로 변별력 있는 키워드 산출.
- **bigram/명사구** 추출 추가(연속 명사 결합).
- 불용어 사전 대폭 확장 + 도메인 불용어(갤 특화) 분리.
- 연관어를 **PMI 기반**으로 교체 → 흔한 단어 상위 독식 제거.

### C. 트렌드 / 이슈 탐지 (신규 축)
- **일자별 키워드 버스트**: 전일(또는 이동평균) 대비 급상승 단어 → "오늘의 떡밥".
- **신규 등장 키워드**: 이전엔 없다가 새로 뜬 단어.
- **감성 시계열**: 날짜·시간별 여론 분위기 곡선.
- **요일 × 시간 활동 히트맵**.
- **참여도 시계열**: 조회·추천·댓글 합계 추이.

### D. 작성자 / 참여 분석 (신규 축)
- 헤비 유저 랭킹, 신규/복귀 작성자, 작성자별 평균 반응.
- **논쟁글 스코어**: `min(추천,비추)/max(...)`·댓글 폭증 결합 → 이슈글 자동 부상.
- 참여 편중도(지니계수) — 소수 여론 주도 여부.

### E. 토픽 모델링 (신규 축, 심화)
- 자동 주제 군집(BERTopic 또는 LLM 라벨링) → "요즘 이 갤 주제 지형도".

### F. 여론 요약 (신규 축, 심화)
- LLM으로 **일자별 / 키워드별 여론 요약 카드** 생성("오늘 △△ 업데이트에 불만 다수" 식).

### G. 인프라
- **분석 결과 캐시 테이블**(증분 계산) — 감성·형태소 결과 저장 후 재사용.
- **Scorer/Analyzer 추상 인터페이스** — 감성·요약 백엔드 교체 가능.
- 재현성(모델·프롬프트·버전 기록).

## 3. 단계별 로드맵

### Phase 1 — 로컬·저비용 즉효 (LLM 없이 체감 개선)
1. 키워드 **특징어(TF-IDF/log-odds)** + bigram + 불용어 확장
2. 연관어 **PMI** 전환
3. **이슈 버스트 / 신규 키워드** 탐지
4. **요일×시간 히트맵** + 참여도·감성 시계열
5. 감성 사전 **슬랭 보강 + 강도 + 이모티콘**(A1)

→ *지금 "허접함"의 절반(특징어 없음·이슈 못 봄)을 API 없이 해소.*

### Phase 2 — 여론 품질 도약 (LLM 도입)
6. `Scorer` 추상화 + **LLM 배치 감성/감정/요약 파이프라인**(A3), A1을 폴백으로 유지
7. **분석 캐시 테이블**(증분)
8. 감성 연속점수 기반 시계열·집계 정밀화

### Phase 3 — 심화 인사이트
9. 토픽 모델링(주제 지형도)
10. 작성자/네트워크·논쟁글 탐지
11. 대시보드 신규 카드(버스트·히트맵·요약·토픽) + API 확장

각 Phase는 기존 테스트(pytest)를 깨지 않고 **증분 추가**하며, 새 지표마다 오프라인 테스트를 동반한다.

## 4. 기대 효과 vs 리스크

| 항목 | 기대 효과 | 리스크 / 대비 |
|---|---|---|
| 특징어·PMI | 진짜 화제어가 상위로 → 인사이트 즉상승 | 낮음(순수 계산) |
| 버스트·시계열 | "오늘 뭐가 터졌나" 파악 가능 | 데이터 3일치라 초기엔 표본 부족 → 누적되며 개선 |
| LLM 감성/요약 | 정확도·반어·요약까지 도약 | API 키·비용·재현성 → 폴백·캐시·버전기록으로 방어 |
| 캐시/추상화 | 확장성·속도 확보 | 스키마 추가(마이그레이션은 기존 additive 패턴 재사용) |

## 5. 결정이 필요한 사항

1. **감성 분석 주력 방식**: A3(LLM 주력+폴백) / A2(로컬 모델) / A1(사전 보강만) 중 무엇으로 갈지
2. **착수 범위**: Phase 1만 먼저 / Phase 1~2 연속 / 전체 로드맵 승인
3. (A3 선택 시) 사용할 LLM·API 예산 상한

> 권장: **Phase 1 먼저 착수(LLM 불필요, 즉효)** → 결과 확인 후 **Phase 2에서 A3 도입.**

## 6. 결정 사항 (확정)

- **감성 주력 방식**: **A3 — LLM 주력 + A1(사전) 오프라인 폴백** (Phase 2에서 도입)
- **착수 범위**: **Phase 1 먼저** → 결과 확인 후 Phase 2 진행
- Phase 1은 LLM 불필요. 아래 작업으로 착수한다.

### Phase 1 작업 목록 — ✅ 완료 (브랜치 `feature/analysis-revamp`)
1. ✅ `keywords.py` — 특징어(`salient_words`, TF-IDF) + bigram/명사구 + 불용어 확장
2. ✅ `keywords.related_words` — 단순 동시출현 → **PMI 기반** 전환
3. ✅ `trends.py`(신규) — 일자별 **키워드 버스트 / 신규 등장어** 탐지
4. ✅ `timeseries.py` — **요일×시간 히트맵** + 참여도(조회·추천·댓글) 추이 + 감성 시계열
5. ✅ `sentiment.py` / `sentiment_ko.json` — 커뮤니티 **슬랭 사전 보강 + 강도 가중치 + 이모티콘 규칙**, 연속점수(`score_norm`)+신뢰도 반환
6. ✅ API(`/analysis/heatmap`, `/bursts`, `keywords?method=salient`, `timeseries?kind=sentiment|engagement`) + 대시보드 카드 + pytest(65개 통과)

> **다음: Phase 2** — `Scorer` 추상화 + LLM 배치 감성/감정/요약(A3), 분석 캐시. 착수 시 별도 확정.

### Phase 2 — 진행 중 (LLM 심층 분석)
사용자 요청: *"특정 단어가 든 글들을 전부 LLM으로 분석해 어떤 평가·내용이 있는지 정리"*.
- ✅ `analysis/llm.py` — Anthropic 래퍼(키/모델 env, lazy import, `available()`, `complete_json`, 테스트 주입 훅)
- ✅ `analysis/llm_report.py` — `keyword_report()`: 키워드 매칭 글+댓글 → 문서화 → **map-reduce** → 구조화 리포트(개요/분위기/소주제/긍정/부정/쟁점/대표인용). `llm_reports` **DB 캐시** + `refresh` + `max_posts` 상한
- ✅ API `POST /analysis/llm_report`, `GET /analysis/llm_status` + 대시보드 🧠 카드
- 백엔드: **OpenRouter**(`OPENROUTER_API_KEY`, 기본) 또는 Anthropic(`ANTHROPIC_API_KEY`) 자동 감지 · 모델 `anthropic/claude-sonnet-5`(`DC_LLM_MODEL`로 교체) · `.env` 자동 로드
- ✅ **다중 키워드(OR)**: `에덴 or 오브, 젬` → 하나라도 포함된 글 매칭(`db.load_posts` q가 리스트 허용)
- ✅ **항목별 근거 링크**: 긍정/부정/쟁점/주제 각 항목이 근거 글번호(post_nos)를 갖고, UI에서 원문 링크로 검증 가능
- ✅ JSON 견고화(따옴표 회피 지시 + 토큰 상향 + 1회 자동 복구)
- 남은 후보: 감성 스코어러를 LLM 경로로 교체(현재는 별도 심층 리포트), 감정 다분류, 일자별 요약 카드
